### Bug fixes shipped today
- **Security fix**: `/api/register` was accepting arbitrary `role` /
  `emailVerified` / `kycStatus` / `walletAddress` from the request
  body — privilege escalation surface. Hardened to explicit field
  destructuring; `insertUserSchema` omits privileged columns.
- **Admin seed script**: `scripts/seed-admin.ts` + `scripts/verify-admin.ts`
  (auto-loads `.env`; idempotent on username).
- **Register + login responses redact** `password`, `totpSecret`,
  `emailOtpHash` via `toClientUser()` helper.
- **Multiple duplicate-header fixes** (clipper-dashboard,
  clipper-directory, admin-metrics, admin-refunds, admin-credit-event,
  ComprehensiveAdminDashboard).
- **Cache-clear order bug**: register/login were calling
  `queryClient.clear()` BEFORE `setQueryData(["/api/user"], user)`,
  detaching the AuthProvider's observer chain. Fixed; the
  "Settings hard reload" workaround was also fixed (it was a
  symptom of this bug).
- **Settings button** uses wouter `<Link>` instead of
  `window.location.href` so it's a proper SPA navigation.
- **Landing page**: Founding Creator section added, "No subscription"
  contradiction fixed, clipper section updated to reference the
  leaderboard, AI claim softened (URL submissions skip AI scan).
- **Cookie policy stub** + sitemap.xml updated (added lastmod, dropped
  /auth, added /cookie-policy).
- **Duplicate "Control" nav entry** removed (it pointed to the same
  component as Dashboard for admins).
- **Bot monitoring sidebar disappearing** fixed (wasn't wrapped in
  DashboardLayout).
- **/admin/control had a custom stale nav** (still referenced
  /admin/enterprise from the Phase 1 strip) — dropped the custom
  navigation prop.
- **Error handling**: `throwIfResNotOk` in queryClient.ts now extracts
  `{ message }` from the JSON body and attaches `.status` as a
  property (`HttpError` type). No more `'401: {"message":"..."}'`
  leaking to user-facing toasts/banners.
- **Email verification banner**: refetches `/api/user` on mount to
  bust the stale React Query cache; treats 409 "already verified"
  from the resend endpoint as success (invalidates query, hides
  banner).
- **Social integration "Connect YouTube" → Instagram bug**: hidden
  form field had a DOM-level `value={selectedPlatform}` override that
  bypassed react-hook-form's state, so every connection submitted as
  Instagram (the defaultValues). Fixed with a `useEffect` that calls
  `form.setValue("platform", selectedPlatform)` on every card click.
